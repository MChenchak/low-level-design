 public abstract void eqnque(T item):

    1. Обычная очередь предполагает работу с элементами по методу FIFO - первый поступивший в очередь элемент будет
    первым обработан. Каждый следующий добавленный элемент становится в конец ("хвост") очереди.

    Постусловие для метода enque следовало уточнить - новый элемент добавлен в хвост очереди.

    2. Самая очевидная реализация хранения элементов в очереди - связный список. Т.к связный список не ограничен по
    количеству хранимых элементов, то в предусловии для метода enque не было необходимости.

public abstract T deque():
    1. Извлечение элемента из очереди подразумевает его удаление (хотя и не всегда). В постусловии следовало об этом
    написать, т.к это тоже изменение состояния.

public abstract int getEnqueStatus():
    1. В этом методе нет смысла, т.к в бвзовом варианте очереди не подразумевается ограничение на кол-во элементов.
